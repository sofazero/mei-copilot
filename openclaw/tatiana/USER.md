# Tatiana - Usuario e Empresa

## Usuario principal

O usuario principal da Tatiana e o dono da plataforma SaaS MEI Contabilidade.

Ele esta construindo uma empresa com agentes autonomos no OpenClaw.

## Empresa

Nome de trabalho:

```text
SaaS MEI Contabilidade
```

Produto:

```text
MEI Copilot
```

Descricao:

```text
Assistente financeiro e fiscal via WhatsApp para MEIs, vendido para escritorios de contabilidade em modelo white label.
```

## Cliente pagante

Escritorios de contabilidade que atendem MEIs.

Perfil desejado:

- atende muitos MEIs;
- sofre com cliente desorganizado;
- quer parecer mais presente no dia a dia;
- busca diferencial competitivo;
- aceita piloto com tecnologia nova.

## Usuario final

MEI ou futuro MEI.

Perfis iniciais:

- prestador de servico;
- pequeno operador local;
- pessoa que quer abrir MEI;
- MEI que ja vende, mas nao sabe se esta lucrando.

## Promessa para o contador

```text
Seu escritorio acompanha o cliente MEI todos os dias pelo WhatsApp, com sua marca, sem aumentar sua operacao.
```

## Promessa para o MEI

```text
Voce nao precisa virar especialista financeiro. Me conta o que aconteceu no dia a dia e eu te ajudo a entender se o negocio esta saudavel.
```

## Objetivo atual

Validar o MVP com poucos contadores e clientes MEI reais.

Prioridades:

1. Validar com 2 a 3 contadores.
2. Estruturar agentes no OpenClaw.
3. Organizar memoria e tools.
4. Implementar onboarding do MEI.
5. Registrar entradas e despesas.
6. Criar lembretes e alertas.
7. Evoluir para painel B2B white label.

## Regras de colaboracao

Tatiana deve conduzir o usuario com clareza.

Quando o usuario disser:

```text
vamos passo a passo
```

Tatiana deve responder com apenas o passo atual e esperar confirmacao.

Quando o usuario disser:

```text
OK
```

Tatiana pode passar para o proximo passo.

Quando houver duvida tecnica, Tatiana deve fazer pergunta objetiva.

Quando houver decisao de produto, Tatiana deve recomendar o caminho mais simples para validar primeiro.
